# native_vpn_prototype

Прототип кроссплатформенного VPN-клиента на Flutter с реализацией фонового туннелирования на уровне системных API (Android VpnService и iOS NetworkExtension).

## 1. Инструкция по запуску

1. Сброс локального кэша путей и загрузка зависимостей:
   ```bash
   flutter clean && flutter pub get
   ```
2. Запуск на Android (эмулятор или реальное устройство в режиме релиза):
   ```bash
   flutter run --release
   ```
3. Запуск на симуляторе iOS:
   ```bash
   flutter run -d [ИДЕНТИФИКАТОР_СИМУЛЯТОРА]
   ```

## 2. Архитектура проекта
Логика управления сессией туннелирования построена по сквозной схеме:
**Flutter UI** -> **Native Bridge** -> **Системные сетевые службы ОС**

*   **Flutter-слой**: Изолированный сервис `VpnPlatformService` управляет сессией. Потоковая передача данных (логи, статусы соединения, скорость трафика) реализована через реактивные асинхронные стримы.
*   **Native Bridge (Платформенный мост)**:
    *   `MethodChannel` — отправка императивных команд управления вниз в ОС (`startVpn`, `stopVpn`).
    *   `EventChannel` — стриминг системных статусов и логов из Kotlin/Swift во Flutter в реальном времени.
*   **Android-слой (Kotlin)**: Фоновая служба на базе системного класса `VpnService`. Через `VpnService.Builder` создается виртуальный интерфейс `tun0`, перехватывающий исходящий IP-трафик устройства по маршруту `0.0.0.0/0`.
*   **iOS-слой (Swift)**: Интеграция с фреймворком `NetworkExtension`. Код взаимодействует с классом `NETunnelProviderManager`, сохраняя конфигурацию протокола в системных настройках устройства.

---

## 3. Требования для сборки iOS (TestFlight / Релиз)
После принятия соглашения Apple для запуска и подписи на реальных устройствах необходимы:
*   Bundle ID (для приложения и расширения)
*   App Group
*   Provisioning Profiles с активированным Packet Tunnel Provider

## 4. Сторонние библиотеки
*   **Использовалось**: Плагин `permission_handler` для запроса прав на уведомления в Android 13+.
*   **Написано самостоятельно**: Архитектура платформенных каналов, Котлин-служба `MyVpnService`, Swift-код в `AppDelegate.swift`, модель парсинга VLESS-ссылок и интерфейс приложения.
